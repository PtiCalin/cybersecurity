# CVE References

Annotated notes on Common Vulnerabilities and Exposures.

## Organization

- **by-year/** — CVEs organized chronologically (e.g., `by-year/2024/`)
- **by-category/** — CVEs organized by vulnerability type (e.g., `by-category/injection/`)

## Naming Convention

Format: `cve-[year]-[number].md`

Examples:
- `cve-2024-12345.md`
- `cve-2023-67890.md`

## Content Focus

Each CVE note should include:
- CVSS score and severity
- Affected systems and versions
- Attack vector and exploitability
- Proof of concept (if available and ethical)
- Mitigation and patches
- Defensive detection strategies
- Real-world impact and case studies

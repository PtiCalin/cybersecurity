## Purpose

This catalog serves as:

- **Learning roadmap** — Recommended reading sequences for progressive skill development

## Learning Paths

### Path 1: Absolute Beginner → Security Professional

**Duration**: 12-18 months of dedicated study

1. **Network Security Essentials** (Stallings) — Foundation
2. **Penetration Testing** (Weidman) — Offensive basics
3. **Blue Team Handbook** (Murdoch) — Defensive basics
4. **The Web Application Hacker's Handbook** — Web security specialization
5. **Security Engineering** (Anderson) — Architecture and advanced concepts

**Parallel**: Practice with TryHackMe, HackTheBox, PortSwigger Academy

---

### Path 2: Developer → Secure Developer

**Duration**: 6-9 months

1. **Secure Coding Principles & Practices** — Defensive programming
2. **The Web Application Hacker's Handbook** — Attacker mindset
3. **The Art of Software Security Assessment** — Advanced code review
4. **Security Engineering** (Anderson) — System design

**Parallel**: OWASP Top 10 exercises, secure code review of own projects

---

### Path 3: Security Generalist → Penetration Tester

**Duration**: 9-12 months

1. **The Web Application Hacker's Handbook** — Web exploitation
2. **Penetration Testing** (Weidman) — Methodology
3. **RTFM** — Command reference
4. **Practical Malware Analysis** — Post-exploitation understanding
5. **The Shellcoder's Handbook** — Advanced exploitation (optional)

**Parallel**: OSCP preparation, CTF challenges, bug bounty practice

---

### Path 4: Security Analyst → Incident Responder

**Duration**: 6-8 months

1. **Practical Packet Analysis** — Network forensics
2. **Incident Response & Computer Forensics** — IR methodology
3. **Practical Malware Analysis** — Threat analysis
4. **BTFM** — Quick reference

**Parallel**: SOC simulations, malware analysis labs, forensic challenges

## Reading Strategies

### Active Reading Approach

1. **Preview** — Skim chapter headings, intro, summary
2. **Question** — What do I want to learn from this chapter?
3. **Read** — Active reading with notes
4. **Recite** — Explain concepts in your own words
5. **Review** — Summarize key points, create reference note

### Time Management

| Reading Type              | Time Allocation | Purpose              |
| ------------------------- | --------------- | -------------------- |
| **Theory sections**       | 60-70%          | Deep understanding   |
| **Examples/case studies** | 20-30%          | Application learning |
| **Exercises**             | 10-20%          | Skill verification   |

Don't skip exercises — they verify understanding!

### Note-Taking While Reading

**Minimum per chapter:**

- [ ] 3-5 key concepts with definitions
- [ ] 2-3 practical examples
- [ ] 1-2 important quotes
- [ ] Questions raised or unclear points
- [ ] How this connects to prior knowledge

**Tools:**

- Physical book → Use sticky notes, mark pages
- PDF → Highlight and annotate directly
- E-reader → Use built-in highlights/notes
- Transfer to reference notes weekly

### Academic Access

- **Université de Montréal Library** — Check O'Reilly Learning, SpringerLink, Safari Books Online

- **Course reserves** — Some may be on reserve for specific courses

### Legal Free Resources

- **Authors' websites** — Some provide free editions
- **Publisher previews** — Sample chapters available
- **ArXiv/Research repositories** — Preprints and papers

### Purchase Priorities

When budget is limited, prioritize:

1. 🟢 Essential books for current learning path
2. Books with high re-reference value (Anderson, Stallings)
3. Books with exercises and labs (practical > pure theory)
